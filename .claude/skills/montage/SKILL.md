# Montage Skill

Assemble multiple generated video clips into a single final video using ffmpeg.

## When to Use

- After completing a **long-video pipeline** — all shots generated, user wants final assembly
- When user explicitly asks to **stitch, combine, merge, or montage** multiple video clips
- **NOT** for single video generation — that's `/video-skill`
- **NOT** automatically after every generation — only when user requests or pipeline completes

## When NOT to Use

- User generated 10 separate videos as individual pieces — no stitching needed
- User hasn't finished generating all clips yet
- User wants to edit a single video (trim, crop, etc.)

## Assembly Methods

### 1. Simple Concatenation (cut-to-cut, no transitions)

Best for: sequential shots, episode blocks.

```bash
# Create file list
for f in output/videos/shot_*.mp4; do
  echo "file '$f'" >> output/filelist.txt
done

# Concatenate
ffmpeg -f concat -safe 0 -i output/filelist.txt -c copy output/final.mp4
```

### 2. Crossfade Transitions

Best for: long-video, brand stories, smooth montages.

```bash
ffmpeg -i output/videos/shot_001.mp4 -i output/videos/shot_002.mp4 \
  -filter_complex "[0:v][1:v]xfade=transition=fade:duration=0.5:offset=7.5[v]" \
  -map "[v]" output/transition.mp4
```

Common transitions: `fade`, `wipeleft`, `wiperight`, `dissolve`, `smoothleft`, `smoothright`

### 3. Add Background Music

```bash
ffmpeg -i output/final.mp4 -i bgm.mp3 \
  -c:v copy -c:a aac -shortest output/final_with_bgm.mp4
```

### 4. Multi-clip with Crossfades (3+ clips)

```bash
# For N clips, chain xfade filters:
ffmpeg -i clip1.mp4 -i clip2.mp4 -i clip3.mp4 \
  -filter_complex "\
    [0:v][1:v]xfade=transition=fade:duration=0.5:offset=7.5[v01]; \
    [v01][2:v]xfade=transition=fade:duration=0.5:offset=15[v]" \
  -map "[v]" output/final.mp4
```

### 5. Add Subtitles (SRT)

```bash
ffmpeg -i output/final.mp4 -vf subtitles=output/subs.srt output/final_subtitled.mp4
```

## Pipeline-Specific Assembly

### Long Video
- Read `assembly.json` from output/ — it has shot order, transitions, BGM notes
- Follow the assembly plan exactly
- Default: simple concatenation unless storyboard specifies transitions


## Output

- Final video saved to `output/final.mp4` (or `output/final_epNNN.mp4` for episodes)
- Always verify output exists and is playable before reporting to user
- Report duration, resolution, file size

## Prerequisites

- `ffmpeg` must be installed on the system
- All source clips must exist in `output/videos/`
- All clips should have matching resolution and frame rate (they will if generated by the same pipeline)
